## 9. 正确 API 预测的机制分析

### 9.1 实验设置

从 SFT+DPO数据集 与 EDAPIBench-starcoder2-7b 子集中，筛选出 StarCoder2-7B 在 **teacher-forcing** 下对 API span 内每个 token 均预测正确的样本，构成 ，共 **376 条**，涉及 tensorflow（237）/ pytorch（95）/ sklearn（43）/ seaborn（1）四个库。

对每条样本，**决策 token** 定义为正确 API 名称在其命名空间/导入别名前缀（如 `tf.`、`torch.`、`F.`）之后的第一个 token——例如 `tf.random.categorical` 的决策 token 是 `random` ；不含命名空间前缀的形式（如直接写作 `GaussianMixture`）则以其自身首 token 为决策 token。

以 `decision_prefix`（必要时追加命名空间前缀）做 teacher-forcing 前向，在决策 token 位置提取每层隐藏状态，经 **Logit Lens** 和 **Tuned Lens** 投影至词表空间，记录：

- **top-10 进入层**（`emergence_layer_top10`）：正确 token 排名首次进入前 10 名所在的层；
- **稳定决策深度**（stable decision depth，`saturation_layer`）：正确 token 排名首次降至第 1 名且此后不再变化的层；

---

### 9.2 层分布

| 指标 | Logit Lens | Tuned Lens |
|---|---|---|
| top-10 进入层（均值 / 中位 / P25–P75） | 19.2 / **20** / 18–21 | 3.4 / **0** / 0–1 |
| 稳定决策深度（均值 / 中位 / P25–P75） | 27.9 / **28** / 27–31 | 24.8 / **30** / 23–31 |
| 未达到稳定决策深度的样本数 | 17/376（4.5%） | 54/376（14.4%） |

![fig1_emergence_layer_histogram_after_dot.png](pic/fig1_emergence_layer_histogram_after_dot.png)

Logit Lens 下，正确 token 进入 top-10 的层集中在第 **18–21 层**（32 层网络的 56–66% 深度），全部 376 个样本最终都进入 top-10；稳定决策深度则普遍要到第 27–31 层才达到，与 top-10 进入层相差约 8 层。

Tuned Lens 下出现了更显著的现象：**256/376（68%）的样本在第 0 层（首个 decoder 层输出）就已经把正确的 API 名称排进 top-10**，部分甚至直接排第 1（如 `interpolate`、`divide`）。也就是说，一旦命名空间前缀（如 `torch.`）被确定，Tuned Lens 训练得到的 translator 在最浅层就已经能从残差流中读出"该命名空间下接下来最可能的 API 名称"，但稳定决策深度均值仍达 24.8。

---

### 9.3 库间差异

| library | 样本数 | top-10 进入层均值 | 稳定决策深度均值 |
|---|---:|---:|---:|
| tensorflow | 237 | 18.2 | 27.1 |
| pytorch | 95 | 20.7 | 30.0 |
| sklearn | 43 | 21.0 | 28.3 |
| seaborn | 1 | 23.0 | 31.0 |

![fig2_emergence_by_library_after_dot.png](pic/fig2_emergence_by_library_after_dot.png)

一种解释是：`tf.` 的高频 API（`constant`/`cast`/`random` 等）在预训练语料中出现密度高，模型能更快将候选收敛到这一小子集中；而 pytorch/sklearn 的 API 名称（如 `linalg.solve` vs `linalg.cholesky`）语义更接近、相互区分度更低，需要更多层才能收敛。
